# archive/ — 過程紀錄層（Post-mortem Data Layer）

本目錄保存專案多團隊治理時代（2026-03 ~ 2026-05）的**過程殘留**：治理規則文件、計數器登記、校準報告、研究方法論文件、已完結的實作計畫。共 158 個 md 檔（130 個主檔＋28 個 .tw 雙語雙胞），於 2026-06-12 蒸餾第一波依 [DISTILLATION_LIST.md](../DISTILLATION_LIST.md) 的逐檔判定移入，**原始內容與相對結構未動**（原路徑 `X/Y.md` → `archive/X/Y.md`）。

## 為什麼保留而不刪除

這批文件是 [RETROSPECTIVE.md](../RETROSPECTIVE.md) 所述灌水模式的**被稽核對象原文**——治理時代宣稱了什麼、用什麼語言宣稱，原樣在此。注意證據鏈的精確邊界：**87.5%／96%／210 票這三個頭條數字的一手稽核紀錄（cycle 03-41/03-42 收官文件）存於專案內部、未隨庫發佈**；本目錄保存的是被那兩次稽核所推翻的宣稱原文。外部讀者可由公開 git 歷史對照這裡的宣稱獨立重算 spec-vs-merged gap；投票數據則無法外部複驗。對研究多智能體系統失效模式的人，這仍是罕見的完整田野資料；對其他讀者，它們是噪音——所以移到這裡，不擋導讀路徑。
