## 8. Pareto-Boundary 論證（DARWIN Cross-Cycle Pattern P4）

依 cycle 03-14 R2 §11 Pattern P4 框架（cycle 03-16 R1 §8.4 + R2 §5.2 重新引用）：

> 對 ~225-252 個 canonical 檔案進行 bilingual obligations 之追溯 sweep-back 將耗費 ~5-15 cycles 之專責翻譯 labor + ratification + glossary 穩定化，且無比例之 readership benefit。Forward-only 為 cost-proportionate Pareto boundary。

任何 cycle 03-16+ retrofit 動議必須展示足夠反證以覆寫 Pareto boundary，此將為例外狀況。Retrofit 動議之 chair-rule 將此標準程序化編纂 — 在無例外證據之情況下，主席無實質投票即裁定為不可受理。

## 9. Re-Framed 動議接受 Pattern

若動議提案者希望推進追溯 TW translation 為專案目標但承認不可受理裁決，可重新框定為：

1. **自願性 CYCLE_INTERNAL informational labor allocation**：
   - 以名稱識別之 high-priority pre-cycle-03-15 文件。
   - 撰寫 labor 配置為 cycle-internal task（無合規 binding）。
   - SCRIBE/LINNAEUS 於 G4-folder gate 進行 post-author review。
   - 產出之 TW 姊妹文件依 Rule #78 §78.1 落於 CYCLE_INTERNAL tier（MAY parity）。
2. **重新框定動議遵循正常 R3 程序** — 實質投票、依 MR-11 之 dissent 保留等。

此 pattern 在尊重 MR-12 既有不破壞 hard constraint 之同時，保留優先前向翻譯努力之作者能動性。

## 10. 參考實作（cycle 03-16+ 主席行為指南）

Cycle 03-17+ R3 主席於浮現追溯 TW parity sweep 動議時收到下列操作 checklist：

```
1. 識別動議是否提議對 grandfathered（pre-cycle-03-15）artefact 進行追溯 sweep？
   → YES：進行 step 2
   → NO：動議遵循正常 R3 程序

2. 識別動議是否針對：
   - canonical openstarry_doc/ pre-cycle-03-15
   - Tenets #1-#10
   - MR-1~MR-13
   - ZT-1/2/3
   - Baseline Rules #1-#77
   - ENG-FAB v1.0~v1.7 items
   - 任何 pre-cycle-03-15 BINDING-ratified artefact
   → YES：進行 step 3
   → NO：動議可繼續（例如 CONSULTATIVE-tier proposal）

3. 套用 chair-rule 不可受理裁決：
   - 引用 5-citation chain（上述 §3）
   - 紀錄裁決於 R3 decision log
   - 提供重新框定路徑（上述 §9）
   - **不**對原始追溯形式進行實質投票

4. 若提案者重新框定：
   - 重新框定動議進入正常 R3 程序
   - 實質投票 + 依 MR-11 之 dissent 保留
```
