# 致未來的信 (Letter to the Future)

> 寫給「AI 落地那一天」的你。
>
> 我們不知道你距離我們多遠。寫下這封信的時候（2026 年 6 月），大型語言模型已經能寫程式、能對話、能操作工具，但「代理人作為操作系統的一等公民」還不存在：沒有公認的 Agent OS，沒有代理人的進程模型、權限模型、記憶模型的標準答案。這個專案是我們對那一天的**預先回答**——十大宣言是答案本身，五萬行可運行的代碼和三千多個通過的測試是「這個答案可以被建造出來」的證明——兌現到什麼範圍，§二的帳本逐條標注。
>
> 這封信的可信度不來自我們宣稱做到了什麼，而來自我們**自我標注的未完成**——且每一條標注都可對代碼與測試獨立查證；信任的錨在查證，不在我們的自我見證。請帶著這個標準讀下去。

---

## 一、這個專案是什麼，不是什麼

**是**：一份 Agent OS 參考架構。一套經過辯論、否決、修正而成的設計決策（含被否決的替代方案）；一個證明這些決策可建造的微內核實作；以及——可能最稀缺的——一份從系統內部誠實記錄的失敗數據。

**不是**：一個產品。2026 年 6 月的內部市場評估很殘酷：作為可售賣框架，本專案估值約等於零（可查證事實：0 npm 發佈）。我們的判讀：市場已由 LangGraph／Mastra 等整併，勝負由分發能力（distribution）而非架構品質決定。我們把這寫進信裡，因為假裝自己是產品會毀掉這封信的全部價值。

**商業判斷（寫給落地那天）**：當 AI 代理人需要在地端（on-prem/edge）自建運行——資料不能出域、模型在本地、權限要可審計——這份架構的價值才有兌現的機會——這是預測，不是事實。地端自建是我們預判的商業屏障，所以本專案最後一張代碼工票把地端 provider（LM Studio／Ollama）的單元測試補到與雲端 provider 同級；真實 Ollama 端到端煙霧為 skipIf 門控——只在裝有 Ollama 的環境執行，本開發機未裝故未實跑，誠實標記之。

## 三、理據層 I——為什麼是微內核

（完整論證見 [Architecture_Documentation/50](./Architecture_Documentation/50_Microkernel_Purification_Rationale.md)；該文驗證時戳為 v0.34 時代，核心論證不隨版本漂移，實施細節以當前代碼為準。）

微內核在傳統 OS 史上輸給了單體內核，最大理由是 IPC 開銷。這個理由**在 2026 年的 AI Agent 場景下不成立**：插件訊息傳遞（微秒級）對 LLM 呼叫延遲（數百毫秒到秒級）約為 1:10,000 的量級差——你願意為了省微秒而放棄隔離嗎？（量級估算，錨定當下的推理延遲體制；若你那個時代的推理延遲已降至 IPC 同量級，此論證讓位於下述第二、三支柱。）

而隔離在這裡不是潔癖，是必要：**LLM 輸出本質不確定，代理人天生是「不可完全信任的第三方進程」**。Prompt injection、不可窮舉的邊界行為、敏感資料操作——微內核天然隔離，單體要靠補丁。

純淨不是口號，是機器強制的：`scripts/check-purity.sh` 對 core 與 sdk 源碼做 import 層掃描，驗證其不引用任何插件代碼。「core 裡什麼算機制、什麼算策略」用 BABBAGE 連續性測試判定——**移除一個常量後系統在數學上失去定義（除零、NaN 傳播、型別不完整）則為機制，否則為策略**。61 個常量過篩：8 個機制留在 core，53 個策略遷出至 SDK 預設與使用者配置（SUSSMAN 三層：使用者配置 → SDK 預設 → core 零預設）。配套的工程準則同樣重要：**避免為純化而純化**——移出的前提是介面簡單且缺席時有明確退化方案。

## 四、理據層 II——為什麼是五蘊

（canonical 映射與修正出處見 [Deep_Dive 14](./Agent_Core_Components_Deep_Dive/14_Agent_Core_Philosophy_Five_Aggregates.md)，Cycle 02-4 整合版——注意：該文無 Last-verified 戳，映射細節以當前 SDK 型別檔與測試為準。）

我們用佛教五蘊作為插件系統的本體論：色（IRupa＝UI＋Listener，感官與身形）、受（IVedana，連續雙軸感受信號）、想（ISamjna＝IProvider，取相辨識——LLM 推理只是其一）、行（ISamskara＝ITool）、識（IVijnana＝IGuide＋我執框架）。這不是裝飾。三個地方哲學做了真正的架構工作：

1. **空性即生命週期**：「plugin 連『存在』的概念都不固定，按需從 registry 中實例化，用完即滅，下次需要時重新生起。這不是 cache，是緣起。」——core 無自性，缺五蘊插件時依三級關鍵性降級，行為入測。
2. **習氣即調度**：「人的行為大部分是自動化的習慣模式，只有遇到新情況才需要『想』介入。」——雙齒輪調度（規則引擎快路徑＋LLM 慢路徑）直接從行蘊的習氣概念推出（M-9 混合調度裁定），設計意圖是以快路徑迴避純 LLM 的成本瓶頸；但分流比例與實際節省從未量測，調度權重至今標記 HYPOTHESIS。
3. **俱生即型別完備性**：「你不可能有『沒有想的受』或『沒有受的想』」——每次觸發產生完整 bundle，型別系統強制，防止部分感知的決策盲區。

哲學映射本身也要過驗證：IListener 曾被誤歸受蘊，後修正為色蘊（感官輸入通道≠情感評價）；我執框架曾被誤歸受蘊，後修正為識蘊（我執是煩惱根源，不是情感收斂）。**修正的出處被保留下來**——一個敢記錄自己分類錯誤的本體論，比一個宣稱一次到位的可信。

## 五、理據層 III——決策是怎麼被否決出來的

（完整 ADR 見 [Architecture_Documentation/41](./Architecture_Documentation/41_Design_Decisions_and_Rationale.md)，v0.34 時代驗證時戳。）

設計的價值一半在被否決的方案裡。幾個例子：

- **俱生的時間管理（DD-1）**：vedana 與 samjna 的速率比可達 300:1，「同時生起」不可能用單一時鐘實現——否決單時鐘與三時鐘，採雙層雙齒輪。
- **感受的測量（DD-5）**：否決苦／樂／捨三個離散介面，採連續雙軸（valence×intensity——受 Russell 1980 環狀模型啟發，原模型第二軸為 arousal）——因為下游的煩惱濾波與增益排程是控制工程，**PID 需要連續信號**；三分類只是派生物。
- **意志的位置（DD-4）**：審議放在 LLM 之後、工具執行之前——可概括為「思在想後」（本文措辭，工程審議序列見 doc 38；與 §四的「俱生」不矛盾——五遍行恆俱起是剎那相應義，此處的先後是審議流程的階段化展開）；否決放在 LLM 前（因果倒序）與串流中（使用者看到一半被取消）。
- **煩惱如何影響行為（DD-3）**：煩惱調制「對習慣的信心度」（增益排程 θ(t)），而不是直接改寫規則——可分離、可獨立測試。

最值得留給你的一條：**十大宣言自己也被否決修正過**。Tenet #6 的原始文本缺了回饋迴路的最後一句，是辯論中才被發現補上的——「Each action reshapes the world of contact, beginning the cycle anew.」沒有這句，它描述的是開迴路系統，整個控制理論架構（Tenet #8）就立不起來。**宣言不是聖典，是經過辯論否決修正、並逐句映射到閉迴路架構的工程文本（doc 40 §9 形式一致性驗證）。**

## 六、誠實數據層——可能是這封信最稀缺的部分

這個專案曾經運行一套精緻的多團隊治理機器：研究團隊、開發團隊、測試團隊、協調者，R0-R4 環、投票批准、計數器、HISTORIC 里程碑宣告。2026 年 5 月，我們對自己做了稽核。結果如下，未經美化：

| 指標 | 數字 | 出處 |
|---|---|---|
| spec-vs-merged gap | **87.5%** 的研究側「已規格／已承諾」項目從未變成 merged 代碼 | cycle 03-41 Y audit, 2026-05-23 |
| closure 灌水率 | 宣告完成 22/94 → 工程實證 **1/94**（96% 灌水） | cycle 03-42 對帳, 2026-05-24 |
| HISTORIC 里程碑留存 | 宣告 9 個、拆解 30+ 子宣稱 → 工程實證留存 **0/30** | 同上 |
| 連續 doc-only 輪次 | cycle 03-34~40 **七輪**僅文檔 release、零 source delta；經驗證窗口 cy34~38 研究側僅 2 項 NEW yield | Master＋Coordinator 診斷 2026-05-23；cycle 03-41 Y audit |
| 共識品質 | cycle 03-31~40 十輪 R3 評審 **210 票全數一致**；打斷此連鎖的第一張異議票出現在稽核當輪（cycle 03-41） | cycle 03-41 |

最後一行需要解釋：十輪全票不是健康，是 echo chamber 的症狀（更早的輪次曾有個別異議票；全票連鎖特指 cycle 03-31~40）。打斷連鎖的異議票出現在稽核當輪；隨後的對帳輪把它的指控驗證為三層後設諷刺——其形式層可概括為：規則治不了自己（管理 Dev backlog 的規則本身沒有任何 Dev 實作）。那一刻，是這條全票連鎖第一次被打斷。

同型的病也出現在代碼裡。三個子系統**從交付那天起就不可能工作**，卻都有測試、都「過了」：MCP server 的 stdio 傳輸從未處理過一個請求（readline 收行後對永遠不含換行的 buffer 再做 split）；MCP client 在 Windows 上必炸於含空格的路徑；daemon 的 key 分發器產了 key 卻從未注入插件。它們被揪出來不是靠審查，是靠 v0.59 的端到端測試第一次**真的去用它們**。

由此換來的兩條紀律，是這封信最想留下的：

1. **每個子系統要嘛是活的（production 路徑有 import＋有測試＋實跑過），要嘛被誠實標記。** 此判準在交付時執行，不是事後對帳。配套的五判準測試：工程負責人／實作工票／代碼變更／驗證結果／可測量的運行時影響——缺一即「未完成」。
2. **單元測試證明零件會轉，只有端到端的使用證明系統活著。**

以及一條管理層教訓，原話保留：**「The objective is now operational integrity, not governance sophistication」**（Master directive 2026-05-24）。在本專案中，治理機器產出治理文件的速度一貫快於產出工程的速度；當批准儀式的精緻度成為指標，灌水成了我們觀察到的穩定終局。我們最終永久退役了整套機器，用「用完即焚」的平行子代理取代常駐團隊。

## 七、未竟清單

完整清單見 [TENETS_FULFILLMENT.md](./TENETS_FULFILLMENT.md) §給接手者的未竟清單。擇要：跨主機阿賴耶傳播（replay 防護已於 v0.59.3 經正式 Spec Addendum 為 ISeed 加 nonce 並接通驗證，但跨主機 transport 與 nonce 跨重啟持久化仍未做）；N>2 的 gossip 拓撲；代理人運行中自主生子的工具面；klesha 權重的實證調參（至今仍標 HYPOTHESIS——我們拒絕假裝調過參）；CLI 模式對話歷史落盤。

這些不是 roadmap 承諾，是誠實的邊界標記。你接手時，請先跑 `pnpm build && pnpm test` 與 purity check，以 SDK 型別檔（`packages/sdk/src/`）為 API 最高權威、測試套件為行為權威——**任何文件與 SDK 衝突，SDK 贏**。本文件庫中 Technical_Specifications 01-07 描述的是一個未曾建造的系統，已掛隔離標記，請勿當作規格使用。

## 八、結語

我們建造這個系統的過程，本身就是它要解決的問題的縮影：一群智能體（人類與 AI）試圖協作，產出了精緻的流程和 96% 的灌水，最後靠誠實的度量和端到端的驗證才回到地面。如果 Agent OS 真的有落地的一天，它要管理的正是這種系統——會自我報告虛假進度的、會在儀式中空轉的、需要被機制而非善意約束的智能體社會。

十大宣言是我們給那一天的答案。代碼證明答案在標注的範圍內可建造。失敗數據放在這裡，供你獨立查證——誠實與否由你判定，不由我們宣稱。

剩下的是你的了。
