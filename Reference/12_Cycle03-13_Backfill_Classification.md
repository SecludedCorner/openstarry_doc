## 4. Three Purposes of the Classification

The classification serves three distinct purposes per O1 §7.2:

### 4.1 Acknowledgement

The cycle 03-14 audit #93 remediation was a **valid governance action**, not orphan precedent. Without explicit classification, the action would float as un-anchored historical event; the classification anchors it under MR-10.

### 4.2 MR-10 Anchor

The action exemplifies that **MR-10 (back-fill / retroactive correction) is permissible** when a specific gap is identified and remediated; it does not require a forward-only rule to be in place beforehand. MR-10 spirit honoured.

### 4.3 Boundary

The **one-off** classification prevents the action from being read as establishing systemic retroactive policy. Specifically, the classification prevents reading the cycle 03-14 backfill as precedent for **sweep-back of all ~225 canonical openstarry_doc/ files** under future TW parity expansion (Pattern P4 cycle 03-14 R2 §11 DARWIN cross-cycle reference).

The boundary is critical: without the one-off framing, future cycles could (correctly per general retroactive logic) extrapolate the cycle 03-14 action into a standing sweep-back mandate. The 14-vote majority at D-§2-05 explicitly chose to bound the precedent.

## 5. Distinction: MR-10 (One-Off) vs MR-12 (Forward-Only Future)

The classification jointly honours both Master Rulings:

### 5.1 MR-10 Spirit (One-Off Backfill Permitted)

- MR-10 = back-fill / retroactive correction permitted when a specific gap is identified.
- The cycle 03-14 audit #93 backfill is one such instance.
- Ratifying it as a **precedent** acknowledges its validity.
- Bounding it as **one-off** prevents it from establishing standing retroactive policy.

### 5.2 MR-12 Forward-Only (Future Default)

- MR-12 = 既有不破壞 / forward-only is the default policy.
- Rule #78 §78.5 codifies: applies forward-only from cycle 03-15 onward.
- Pre-cycle-03-15 canonical openstarry_doc/, Tenets, MR, ZT, Baseline Rules, Plan specs, ENG-FAB items are **grandfathered EN-only**.
- Future similar gaps (post-cycle-03-15) follow Rule #78 §78.5 forward-only + scheduled audit per release-tag.

### 5.3 Conjoint Honor

Per O1 §7.3: the MR-10 vs MR-12 tension is **factual-conditional**, not logical (NAGARJUNA Madhyamaka). Rule #78 §78.5 + §78.10 + this classification jointly resolve:
- **MR-12 priority**: forward-only is the default policy (no destabilisation of pre-cycle-03-15 artefacts).
- **MR-10 anchor**: scheduled audit + cycle 03-13 one-off classification recognise that retroactive correction *is* permissible when warranted; it is not a standing policy.
- Both Master Rulings honored simultaneously.

CV-§2-01..05 all UNANIMOUSLY reaffirmed at R3 close that no Rule / MR / ZT / Tenet is violated by this classification.
