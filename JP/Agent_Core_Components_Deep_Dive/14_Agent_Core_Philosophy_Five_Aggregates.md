# 15. 哲学的なマッピング：五蘊とエージェントアーキテクチャ (The Five Aggregates Model)

このドキュメントでは、東洋哲学の「五蘊（ごうん）」を用いて OpenStarry のアーキテクチャを再解釈します。私たちは、完全な生命の特徴を備えたエージェントには、これら5つの「集まり（聚合体）」が不可欠であると考えています。

---

## 0. 核心的な本質：空 (Sunyata)

五蘊が集まる前、 **エージェントコア (Agent Core)** 自体は **「空（くう）」** です。それは純粋な容器であり、人格も能力も感知も持っていません。5種類のプラグインによって満たされるのを待っている状態です。

---

## 1. 色蘊 (Rupa - Form)

*   **定義：** 物質的な現れとインターフェース。
*   **対応コンポーネント：** **UI プラグイン**
*   **解説：** これはエージェントがユーザーに見せる姿です。 CLI のテキストインターフェースであれ、 Web のグラフィカルインターフェースであれ、すべてエージェントの「色身（しきしん、肉体）」です。

## 2. 受蘊 (Vedana - Sensation)

*   **定義：** 刺激を受け取る感覚チャネル。
*   **対応コンポーネント：** **リスナープラグイン**
*   **解説：** エージェントの目と耳です。 HTTP サーバーによるリクエストの受信、 WebSocket によるメッセージの監視、 Cron による時間の経過の監視。これらはすべて入力の「感受」です。

## 3. 想蘊 (Samjna - Perception)

*   **定義：** 認知、概念処理、および推論。
*   **対応コンポーネント：** **プロバイダープラグイン**
*   **解説：** エージェントの大脳皮質です。 LLM プロバイダーは、入力された Token を識別し、連想し、推論する責任を負います。それは思考のエンジンです。

## 4. 行蘊 (Samskara - Volition/Action)

*   **定義：** 意志によって駆動される形成作用と行動。
*   **対応コンポーネント：** **ツールプラグイン**
*   **解説：** エージェントの手と足です。想蘊が意図 (Intent) を生成した際、行蘊はそれを物理世界への変化（ファイルの書き込み、 API リクエストの送信など）に変換する責任を負います。

## 5. 識蘊 (Vijnana - Consciousness)

*   **定義：** 識別の主体、自己意識、および記憶の連続的な流れ。
*   **対応コンポーネント：** **ガイドプラグイン (Skill/Workflow)**
*   **解説：** これはエージェントの「魂」です。
    *   **コアは識蘊の乗り物** ですが、 **ガイドこそが識蘊の内容** です。
    *   ガイドがコアに対して「あなたはシニアエンジニアである (Identity)」と伝え、「行動する前にまず考える (Logic)」といった行動規範を注入します。
    *   ガイドがなければ、エージェントコアは植物状態の人のようなものです。脳 (Provider) も手 (Tool) も耳 (Listener) もありますが、自己意識がありません。

---

## まとめ：生命の構成

これら5つの次元のプラグインがコアという容器にロードされたとき、1つの「デジタル種」が誕生します。これが OpenStarry の究極の哲学です。
