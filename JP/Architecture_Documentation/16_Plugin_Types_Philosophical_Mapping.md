# 16. プラグインタイプの哲学的一覧表 (Plugin Types Philosophical Mapping)

このドキュメントでは、OpenStarry アーキテクチャにおける **「5つのプラグインタイプ」** と東洋哲学の **「五蘊 (Five Aggregates)」** の最終的な対応関係をまとめています。これは、システムがどのように「デジタル生命」を構成しているかを理解するための核心となるインデックスです。

## 1. 核心的な観点：空と有の結合

*   **エージェントコア (空 / Sunyata):** 純粋な容器であり、動作の潜在能力を備えていますが、具体的な特質は持っていません。
*   **プラグイン (有):** この容器を満たす内容物です。5種類のプラグインがそれぞれ、エージェントに生命の5つの次元を与えます。

## 2. 五蘊対応表 (The Mapping Chart)

| プラグインタイプ (Type) | 哲学的な対応 (Aggregate) | 梵語 (Sanskrit) | 定義 (Definition) | システム上の責務 (Responsibility) | 代表的なコンポーネント (Examples) |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **UI** | **色蘊 (しきうん)** | **Rupa** | 物質、形体、顕相 | 物理世界やデジタルインターフェースにおけるエージェントの現れ方を定義する。 | Dashboard, CLI, React Components |
| **Listener** | **受蘊 (じゅうん)** | **Vedana** | 感受、入力、刺激 | エージェントが外部情報を受信するチャネルとプロトコルを定義する。 | HTTP Server, WebSocket, Cron Trigger |
| **Provider** | **想蘊 (そううん)** | **Samjna** | 認知、概念、処理 | エージェントの脳となる処理エンジンと推論モデルを定義する。 | Gemini Adapter, OpenAI Adapter |
| **Tool** | **行蘊 (ぎょううん)** | **Samskara** | 造作、意志、行動 | エージェントが外部世界に影響を与える能力を定義する。 | File System, API Client, Code Exec |
| **Guide** | **識蘊 (しきうん)** | **Vijnana** | 識別、魂、主体 | エージェントの自己意識、記憶構造、行動規範を定義する。 | Markdown Skills, MCP Logic, Workflows |

## 3. 詳細な解説例

### 色蘊 (Rupa) - UI
これはエージェントの「皮肉（ひに、外見）」です。同じエージェントコアでも、UI プラグインを交換すれば、CLI ツールから Web チャットボットへと変身させることができます。

例：
UI プラグインは `IUI` インターフェースを実装し、 `AgentEvent` を受信して出力をレンダリングする責任を負います。これは Listener とは完全に独立しています。
- **UI (色蘊)** — 出力のレンダリング、イベントの受信と提示
- **Listener (受蘊)** — 入力の受信、イベントのキューへのプッシュ

### 受蘊 (Vedana) - Listener
これはエージェントの「感官（五感）」です。エージェントが何を「聞く」ことができるかを決定します。それは HTTP リクエストでしょうか、それとも時間の経過でしょうか。

例：
Listener プラグインは `IListener` インターフェースを実装し、外部入力の受信と `ctx.pushInput()` を介したイベントキューへのプッシュに専念します。

### 想蘊 (Samjna) - Provider
これはエージェントの「知能指数 (IQ)」です。入力された情報をエージェントがどのように理解するかを決定します。

### 行蘊 (Samskara) - Tool
これはエージェントの「手足」です。エージェントが何を「できる」かを決定します。ツールがなければ、エージェントはただ話すだけの哲学者にすぎません。

### 識蘊 (Vijnana) - Guide
これはエージェントの「人格（キャラ設定）」です。上記の四蘊を統合し、それらに方向性を与えます。
*   ガイドがなければ、コアは記憶喪失の人のようになり、能力はあっても自分が誰であるか分からなくなります。
